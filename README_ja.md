# Modularity Skills（日本語版）

**要約:** [Balanced Coupling（均衡結合）](https://coupling.dev)モデルを使用してモジュラソフトウェアシステムを設計・分析するための [Claude Code](https://code.claude.com) プラグイン。

AIツールにはコードレベルのフィードバックを提供するものが数多くある: ベストプラクティス、エッジケース、潜在的なバグ。それは有用だが、コストのかかるミスが隠れている場所ではない。AIの時代では、コードはかつてないほど速く生成され、技術的負債の蓄積も速くなっている。アーキテクチャ上の非効率、誤って引かれた境界、管理されていない結合はどれも、以前は不可能だったペースで大きな泥の玉へと成長していく。

このプラグインはアーキテクチャレベルで動作する。2つのスキルを含む:

- **`/modularity:review`** は、既存のコードベースの結合不均衡を分析する: 何の知識が適切にカプセル化されているか、何がコンポーネント境界を越えて漏洩しているか、どこに連鎖変更が待ち構えているか。モジュラリティ改善のための実行可能な推奨事項を生成する。
- **`/modularity:design`** は逆方向に進み、機能要件からモジュラアーキテクチャを設計し、統合コントラクト、テスト仕様、完全な結合評価を含むモジュール設計ドキュメントを生成する。

両スキルは [Balanced Coupling](https://coupling.dev) モデルに根拠づけられており、すべての推奨事項は直感ではなく具体的な次元（統合強度、距離、変動性）に基づいている。

> _「これまでのバイブコーディングプロジェクトに対して最も有用な Claude Code スキルの一つ！」_ — Alexei Ledenev（[pumba](https://github.com/alexei-led/pumba)、[k8s-mcp-server](https://github.com/alexei-led/k8s-mcp-server)、[ccgram](https://github.com/alexei-led/ccgram)）

## インストール

[Claude Code](https://code.claude.com/docs/en/quickstart) v1.0.33 以降が必要。

マーケットプレイスを追加してプラグインをインストール:

```
/plugin marketplace add vladikk/modularity
/plugin install modularity@vladikk-modularity
```

または、リポジトリをクローンして直接ロード:

```bash
git clone https://github.com/vladikk/modularity.git
claude --plugin-dir ./modularity
```

## スキル

### `/modularity:review` — モジュラリティレビュー

既存のコードベースの結合不均衡を分析する。以下のときに使用:

- 結合問題のためのコードレビュー
- アーキテクチャ品質の評価
- 分散モノリストのリスクの特定
- ある領域の変更が他の場所に問題を起こす理由の調査

**動作方法:**

1. コードと機能要件ドキュメントを読む
2. ドメイン分類（コア/サポーティング/ジェネリック）、チーム構造、既知の問題点について質問する
3. 3つの次元にわたってコンポーネント間の統合をマッピングする: 統合強度、距離、変動性
4. バランスルールを適用して不均衡にフラグを立てる
5. 結合概要テーブル、問題の説明、具体的な改善推奨事項を含むレビュードキュメント（Markdown + HTML）を生成する

**使用方法:**

```
/modularity:review
```

レビュー出力には、参照される各結合概念に対して [coupling.dev](https://coupling.dev) へのハイパーリンクが含まれる。

### `/modularity:design` — モジュラ設計

機能要件からモジュラアーキテクチャを設計する。以下のときに使用:

- ゼロから新しいシステムを設計するとき
- 既存システムのアーキテクチャドキュメントを作成するとき
- 統合コントラクトとテスト仕様を含むモジュールレベルの設計仕様を生成するとき

**動作方法:**

1. 機能要件を読み、明確化のための質問をする
2. ドメイン領域を変動性によって分類する（コア/サポーティング/ジェネリックサブドメイン）
3. 3つの次元すべてにわたって結合を評価しながらモジュラアーキテクチャを設計する
4. 各モジュールの設計ドキュメントを生成する（責務、カプセル化された知識、統合コントラクト、変更ベクター）
5. 各モジュールのテスト仕様を生成する
6. アーキテクチャ概要ドキュメントを生成する
7. モジュラリティ不均衡について設計を自己レビューし、クリーンになるまで繰り返す

各ステップには進む前の承認が必要だ。

**使用方法:**

```
/modularity:design
```

## Balanced Coupling（均衡結合）モデル

両スキルは3つの次元にわたって結合を評価する Balanced Coupling モデルに根拠づけられている:

- **統合強度（Integration Strength）** — コンポーネント間で共有される知識の量（Intrusive（侵入的）> Functional（機能的）> Model（モデル）> Contract（コントラクト））
- **距離（Distance）** — コンポーネントを共に進化させるソシオテクニカルなコスト（コード構造、チーム境界、ランタイム依存関係）
- **変動性（Volatility）** — ビジネスドメインの観点から評価された、コンポーネントが変更を必要とする確率

バランスルール: `BALANCE = (STRENGTH XOR DISTANCE) OR NOT VOLATILITY`

結合は、強度と距離が互いにバランスをとるとき、または不均衡を中和するほど変動性が低いときに均衡している。

詳細は [coupling.dev](https://coupling.dev) で。

Claude Opus 4.5 以降。このスキルは、最も能力の高いモデルから恩恵を受ける微妙なアーキテクチャ上の推論に依存する。
